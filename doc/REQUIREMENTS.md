# 需求規格書

## 1. 功能需求

### FR-1 左側抽獎項目面板

| ID | 描述 |
|----|------|
| FR-1.1 | 可上傳**一張**項目圖片（拖放或點擊選檔） |
| FR-1.2 | 可編輯項目標題文字 |
| FR-1.3 | 抽獎進行中**不需**切換項目；要換項目時由使用者點擊左側自行更新圖片或文字 |
| FR-1.4 | 圖片僅存於瀏覽器記憶體（data URL），不上傳伺服器 |

### FR-2 右側幸運轉盤

| ID | 描述 |
|----|------|
| FR-2.1 | 轉盤扇區來自「抽獎名單」（右下輸入區解析結果） |
| FR-2.2 | 點擊轉盤（或明確的開始按鈕）開始旋轉 |
| FR-2.3 | 旋轉結束後，指針所指扇區即為得獎者 |
| FR-2.4 | **已得獎者**不再出現在轉盤扇區中 |
| FR-2.5 | 旋轉中禁止重複觸發抽獎 |
| FR-2.6 | 名單皆已得獎或名單為空時，提示無法抽選 |

### FR-3 得獎展示

| ID | 描述 |
|----|------|
| FR-3.1 | 得獎後顯示彈跳視窗（Modal），展示得獎者姓名與目前項目標題 |
| FR-3.2 | 彈窗出現時播放煙火動畫（canvas-confetti） |
| FR-3.3 | 可選開啟慶祝音效（預設關閉，需使用者開啟） |
| FR-3.4 | 關閉彈窗後可繼續下一輪抽獎 |

### FR-4 下方雙欄文字輸入

| ID | 描述 |
|----|------|
| FR-4.1 | **左欄**：抽獎項目文字（可多筆，供參考或快速帶入標題） |
| FR-4.2 | **右欄**：抽獎名單（轉盤扇區來源） |
| FR-4.3 | 兩欄內容皆可能是人名或獎項名稱，由使用者自行定義 |
| FR-4.4 | 以**空格**或**換行**分隔多筆內容 |
| FR-4.5 | 輸入即時解析並更新轉盤（旋轉中除外） |

### FR-5 贊助連結

| ID | 描述 |
|----|------|
| FR-5.1 | 頁尾顯示「請我喝杯咖啡」連結，開新分頁外連至贊助平台 |

### FR-6 資料生命週期

| ID | 描述 |
|----|------|
| FR-6.1 | 不使用後端 API |
| FR-6.2 | 不使用 localStorage / IndexedDB 等持久化 |
| FR-6.3 | 關閉或重新整理分頁後，所有狀態重置 |

## 2. 非功能需求

| ID | 描述 |
|----|------|
| NFR-1 | 單頁即可完成例會抽獎流程 |
| NFR-2 | 支援桌面投影與平板（響應式版面） |
| NFR-3 | 轉盤動畫流暢（目標 ≥ 30fps） |
| NFR-4 | 單張上傳圖片建議上限 2MB，超過時提示 |
| NFR-5 | 靜態匯出後可離線使用（除需外連的贊助 URL） |
| NFR-6 | 繁體中文介面 |

---

## 3. 邊界條件與預期行為

| 情境 | 預期行為 |
|------|----------|
| 名單為空 | 轉盤顯示提示，無法開始旋轉 |
| 名單僅 1 人且未得獎 | 轉盤單扇區，仍可旋轉並開獎 |
| 名單全部已得獎 | 提示「無可抽選對象」，可重置得獎紀錄 |
| 重複名稱 | 以內部唯一 id 區分，排除得獎時不誤刪他人 |
| 旋轉中修改名單 | 建議鎖定輸入或延遲套用至旋轉結束 |
| 圖片格式不支援 | 提示僅接受 image/* |
| 瀏覽器阻擋音效 | 靜默失敗，不影響抽獎流程 |

---

## 4. 不在本期範圍（Out of Scope）

- 使用者帳號、登入
- 伺服器儲存、雲端同步
- 自動依項目清單切換獎項（需手動更新左側）
- 正式 BNI 商標素材（待設計圖提供）
